# 부부공동명의(분양권 전매신청) 예약 시스템 - 설치 가이드

인천검단 AB13 호반써밋Ⅲ 안내문 기준으로 만든 예약 시스템입니다.
구조는 기존에 쓰시던 양재/구미 시스템과 동일하게
**Google Apps Script(백엔드+DB+메일) + GitHub Pages(프론트엔드)** 조합입니다.

## 파일 구성
- `Code.gs` → Google Apps Script 프로젝트에 붙여넣기 (백엔드)
- `index.html` → 신청자용 예약 페이지 (GitHub Pages에 업로드)
- `admin.html` → 관리자용 일정/신청내역 관리 페이지 (GitHub Pages에 업로드, 비밀번호로 보호)
- `guide.html` → 1~2페이지 안내문·절차/구비서류 안내를 보여주는 웹페이지 (GitHub Pages에 업로드)
- `check.html` → 신청자 본인이 동/호·계약자 성명·휴대폰 뒷자리 4자리로 접수확인 하는 페이지 (GitHub Pages에 업로드)
- `부부공동명의_안내문.pdf` → 원본 안내문 PDF. `guide.html`의 "PDF 다운로드" 버튼이 참조하므로
  **반드시 `guide.html`과 같은 폴더에 파일명 그대로 업로드**해야 다운로드 버튼이 정상 동작합니다.
  (파일명을 바꾸고 싶다면 `guide.html` 안의 두 곳 `href="부부공동명의_안내문.pdf"`도 함께 수정)

## 1) Google 스프레드시트 준비
1. 새 스프레드시트 생성 (예: "AB13_전매신청_DB")
2. URL의 `/d/`와 `/edit` 사이 값이 스프레드시트 ID → `Code.gs`의 `SPREADSHEET_ID`에 입력
3. 시트 탭(`슬롯`, `예약`)은 처음 실행 시 자동 생성되므로 미리 만들 필요 없음

## 2) 첨부파일 저장용 Drive 폴더
1. Google Drive에 폴더 생성 (예: "AB13_전매신청_첨부파일")
2. 폴더 URL의 마지막 부분이 폴더 ID → `Code.gs`의 `DRIVE_FOLDER_ID`에 입력

## 3) Apps Script 배포
1. 스프레드시트 > 확장 프로그램 > Apps Script
2. 기본 `Code.gs` 내용을 전부 지우고 첨부된 `Code.gs` 내용 붙여넣기
3. 상단 CONFIG에서 다음 값 수정
   - `SPREADSHEET_ID`, `DRIVE_FOLDER_ID`
   - `NOTIFY_EMAIL` (현재 16701331@hobanenc.co.kr 로 기본 설정됨, 필요시 변경)
   - `ADMIN_PASSWORD` (관리자 페이지 비밀번호로 원하는 값 설정)
4. 저장 후 `배포 > 새 배포`
   - 유형: 웹 앱
   - 실행 계정: 나(본인)
   - 액세스 권한: **모든 사용자** (Google 로그인 없이도 접근 가능해야 함)
5. 배포 후 나오는 웹 앱 URL(`https://script.google.com/macros/s/xxxx/exec`) 복사

## 4) GitHub Pages 설정
1. GitHub 저장소 생성 (예: `ab13-jeonmae`)
2. `index.html`, `admin.html`, `guide.html`, `check.html`, `부부공동명의_안내문.pdf` 업로드 (모두 같은 폴더)
3. `index.html`, `admin.html`, `check.html` 안의 `API_URL` 값을 3번에서 복사한 웹앱 URL로 교체 (`guide.html`은 API를 쓰지 않으므로 수정 불필요)
4. 저장소 Settings > Pages 에서 GitHub Pages 활성화 (main 브랜치 / root)
5. 발급되는 주소(예: `https://아이디.github.io/ab13-jeonmae/`)가 신청자용 페이지
   - 관리자 페이지는 `.../admin.html` 로 접속

## 5) 코드 수정 시 재배포 방법 (URL 유지)
Apps Script 코드를 수정한 뒤에는 "새 배포"를 다시 만들지 말고,
`배포 > 배포 관리 > 연필(수정) 아이콘 > 버전: 새 버전 > 배포` 로 갱신해야
기존 웹앱 URL이 그대로 유지되어 GitHub Pages 쪽을 다시 고치지 않아도 됩니다.

## 사용 흐름
- **관리자**: `admin.html` 접속 → 비밀번호 로그인 → 기간/시작·종료시간/점심시간/슬롯당 정원 입력 후
  "일정 생성" → 30분 단위로 자동으로 예약 가능 시간대가 만들어짐
- **신청자**: `index.html` 접속 → (상단 링크로 `guide.html`에서 안내문 내용 확인 및 PDF 다운로드 가능) →
  원하는 날짜/시간 선택 → 전매신청서 정보 입력 →
  가족관계증명서 첨부 → 접수하면 지정된 메일(`NOTIFY_EMAIL`)로 신청 내용과 첨부파일이 자동 발송되고,
  스프레드시트 `예약` 시트에도 기록됨
- **안내문 페이지**: `guide.html`은 안내문 1~2페이지(신청 안내, 절차 및 구비서류) 내용을 웹페이지로 보여주며,
  원본 PDF를 그대로 다운로드할 수 있는 버튼도 포함되어 있음
- **접수확인**: `check.html`에서 신청자가 동/호, 계약자(양도인) 성명, 휴대폰 번호 뒷자리 4자리를 입력하면
  해당 조건과 정확히 일치하는 접수 건의 접수일시·방문예정일시·상태를 조회할 수 있음
  (개인정보 보호를 위해 4가지 값이 모두 일치해야만 조회되며, 주소 등 상세정보는 노출하지 않음)
- **접수 상태 변경**: `admin.html` 접수 내역 표의 "상태" 열에서 드롭다운으로
  접수완료 → 서류심사중 → LH동의완료 → 계약완료(또는 반려) 로 상태를 바꾸면
  신청자가 `check.html`에서 조회 시 바로 반영됨

## 참고 (CORS 관련)
- POST 요청은 `Content-Type: text/plain`으로 보내 브라우저의 CORS Preflight(OPTIONS)를
  발생시키지 않도록 처리되어 있습니다. (기존에 겪으셨던 Apps Script CORS 이슈 회피 방식과 동일)
- GET 요청(`getSlots`)은 커스텀 헤더 없이 단순 요청이라 별도 처리 없이 동작합니다.

## 커스터마이징 참고
- 안내문 상의 "매월 둘째주 화요일"처럼 특정 요일만 열고 싶다면
  관리자 페이지의 "요일 제한" 옵션에서 화요일만 선택 후 시작일~종료일 범위를 넉넉히 잡아 생성하면 됩니다.
- 점심시간(11:30~13:00)은 기본값으로 들어가 있으며 필요시 값만 바꾸면 됩니다.
- 신청서 항목(동/호, 양도인/양수인 정보)은 3페이지 분양권 전매신청서 양식 기준이며,
  실제 안내문 항목이 추가로 필요하면 `index.html`의 폼과 `Code.gs`의 `RESV_HEADERS`/`handleReserve_`에
  같은 방식으로 필드를 추가하면 됩니다.
